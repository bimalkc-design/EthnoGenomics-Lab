Research

I lead research in organellar genome mining of medicinal and herbal plants, combining field-collected vouchers, classical taxonomy, molecular lab work, and bioinformatics analysis pipelines. My approach emphasises reproducible workflows and integration of traditional ecological knowledge where appropriate.

Ongoing themes

  • Organellar genome assembly & annotation (chloroplast, mitochondrial) from short- and long-read data
  • Comparative genomics for medicinal plant authentication
  • Development of bioinformatics pipelines and scripts for organellar genome mining
  • Ethnobotany and community knowledge integration
  • GHG inventory related to vegetation and land use (method development)

Representative methods

DNA extraction, library preparation (Illumina & Oxford Nanopore), hybrid assembly, annotation with GeSeq / PGA / MITOS, phylogenomics, custom scripts (Python / R) for automated analyses, and data deposition to NCBI/ENA.

Setup & Deploy (GitHub Pages)

Follow these steps to publish this site on GitHub Pages (recommended and free):

  1. Create a GitHub account if you don't have one: github.com
  2. Make a new repository. If you want the site at https://<username>.github.io then name the repo <username>.github.io. Otherwise any repo name is fine (site will be at https://<username>.github.io/<repo>/).
  3. Upload this file as index.html. Also upload assets (portrait.jpg, cv.pdf, gallery images, scripts) into the repo.
  4. In the repository settings > Pages: select branch main (or gh-pages) and folder / (root). Save — your site will publish within minutes.
  5. Optional: connect a custom domain via the same Pages settings. GitHub provides HTTPS by default.

Local preview

Run a quick local server to preview:

python3 -m http.server 8000
# then open http://localhost:8000 in your browser

Alternative: Netlify (drag & drop)

Zip the files (index.html + assets) and drag to Netlify Drop for immediate hosting with HTTPS and a managed domain. Connect your GitHub repo for continuous deploy.

Security & privacy notes

Avoid publishing sensitive personal data (personal phone, ID numbers). For institutional email use the official address. If you host datasets with sensitive information, consider controlled access or DOI with repository (Zenodo, Figshare).
